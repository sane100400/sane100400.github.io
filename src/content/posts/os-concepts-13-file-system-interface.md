## 파일은 저장장치의 사용자 친화적 추상화

사용자는 디스크 블록 번호를 직접 기억하지 않는다. 파일 시스템은 데이터를 이름 있는 파일로 보여 주고, 파일에 대한 open, read, write, close 같은 연산을 제공한다. 파일에는 내용뿐 아니라 이름, 식별자, 위치, 크기, 보호 정보, 시간 정보 같은 메타데이터가 붙는다.

열린 파일은 파일 이름보다 파일 디스크립터나 핸들로 다뤄진다. 이름은 찾기 위한 정보이고, 열린 뒤에는 커널이 관리하는 열린 파일 테이블의 항목이 중요해진다.

## 접근 방법

순차 접근은 앞에서부터 차례대로 읽거나 쓰는 방식이다. 로그 파일이나 스트림 처리에 잘 맞는다. 직접 접근은 특정 위치의 블록이나 레코드로 바로 이동해 읽는 방식이다. 데이터베이스나 큰 파일의 일부만 수정하는 작업에 필요하다.

접근 방법은 파일 시스템 내부 구현과 캐싱 정책에도 영향을 준다. 운영체제는 순차 접근에서는 미리 읽기를 할 수 있고, 랜덤 접근에서는 다른 최적화가 필요하다.

## 디렉터리와 경로

디렉터리는 파일 이름을 파일의 내부 식별자나 메타데이터로 연결하는 자료구조다. 단일 디렉터리는 단순하지만 이름 충돌과 관리 문제가 크다. 트리 구조 디렉터리는 사용자별, 프로젝트별로 파일을 나눠 관리하기 쉽다.

경로 해석은 루트 또는 현재 디렉터리에서 시작해 각 이름 요소를 차례대로 찾는 과정이다. 심볼릭 링크가 있으면 경로 해석이 다른 경로로 이어질 수 있다. 따라서 디렉터리는 화면의 폴더 아이콘이 아니라 이름 공간을 관리하는 기준이 되는 구조다.

## 파일 보호

여러 사용자와 프로세스가 같은 파일 시스템을 쓰면 권한이 필요하다. 읽기, 쓰기, 실행, 삭제, 소유자 변경 같은 연산을 사용자나 그룹별로 제한한다. UNIX의 rwx 비트는 단순한 모델이고, ACL은 객체별로 더 세밀한 권한 목록을 둘 수 있다.

파일 보호는 16장 보안과 17장 보호의 구체적인 사례다. 이름을 찾을 수 있다는 것과 내용을 읽을 수 있다는 것은 다른 문제이며, 실행 권한도 읽기 권한과 별개로 판단해야 한다.

## 예시: 파일 이름과 열린 파일의 차이

1. 프로그램이 `/home/a.txt`를 열 때는 디렉터리를 따라가며 이름을 inode나 FCB로 해석한다.
2. 한 번 열고 나면 프로그램은 경로 문자열 대신 파일 디스크립터를 사용한다.
3. 파일 디스크립터는 열린 파일 테이블을 가리키고, 그 안에는 현재 offset과 접근 모드가 저장된다.

## 용어 정리

### 파일

- 의미: 이름이 붙은 데이터 저장 단위.
- 기능: 바이트 배열 또는 레코드 집합을 일관된 방식으로 읽고 쓴다.
- 주의: 파일 이름은 경로 해석의 입력이고, 열린 뒤에는 파일 디스크립터를 봐야 한다.
- 키워드: `file`, `descriptor`

### 파일 속성

- 의미: 이름, 식별자, 위치, 크기, 권한, 시간 정보.
- 기능: 파일 관리와 보호에 필요한 메타데이터를 제공한다.
- 주의: 속성은 파일 내용과 별개로 관리된다.
- 키워드: `metadata`

### 접근 방법

- 의미: 파일 데이터를 읽는 방식.
- 기능: 순차 접근은 스트림에, 직접 접근은 블록/레코드 기반 처리에 적합하다.
- 주의: 접근 패턴은 파일 시스템 성능에 영향을 준다.
- 키워드: `sequential`, `direct`

### 디렉터리

- 의미: 이름을 파일 메타데이터에 연결하는 자료구조.
- 기능: 경로 탐색, 파일 그룹화, 이름 충돌 방지를 담당한다.
- 주의: 폴더 UI가 아니라 이름 매핑 구조로 이해해야 한다.
- 키워드: `directory`, `path`

### 파일 보호

- 의미: 파일 연산 권한을 통제하는 방식.
- 기능: read, write, execute, delete 등을 사용자나 그룹별로 제한한다.
- 주의: 실행 권한과 읽기 권한은 같은 뜻이 아니다.
- 키워드: `permission`, `ACL`

## 조심할 부분

- 파일 이름은 디렉터리에서 찾기 위한 이름이고, 열린 뒤에는 파일 디스크립터가 중심이 된다.
- 순차 접근과 직접 접근은 성능 특성이 다르다.
- 실행 권한과 읽기 권한은 같은 뜻이 아니다.
