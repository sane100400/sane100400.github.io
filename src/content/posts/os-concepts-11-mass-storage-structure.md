`Operating System Concepts` 10판 Chapter 11, **Mass-Storage Structure** 정리다.

공룡책 10판 기준 PDF p.573 근처에서 시작한다. HDD, SSD, NVM의 물리적 특성과 성능 차이부터 디스크 스케줄링 알고리즘이 줄이려는 비용까지 다룬다.

## 먼저 볼 부분

- HDD, SSD, NVM의 물리적 특성과 성능 차이
- 디스크 스케줄링 알고리즘이 줄이려는 비용
- 오류 검출·수정과 저장장치 신뢰성
- 스왑 공간과 RAID 구조

## 저장장치는 메모리와 다른 병목을 가진다

메인 메모리는 바이트 단위 접근에 가깝지만, 대용량 저장장치는 블록 단위로 접근한다. HDD는 회전하는 원판과 이동하는 헤드가 있어 seek time과 rotational latency가 성능을 좌우한다. SSD와 NVM은 기계적 움직임이 없으므로 랜덤 접근 지연이 낮지만, 쓰기 증폭과 마모 관리 같은 다른 문제가 있다.

운영체제는 장치 특성을 모르면 좋은 정책을 만들 수 없다. 같은 I/O 요청이라도 HDD와 SSD에서 최적화 방향이 달라진다.

![대용량 저장장치 - 저장장치 성능 요소](/os-concepts/diagrams/os-11-mass-storage-structure-detail.png)

> 그림 읽기: HDD는 헤드 이동과 회전 지연이 크고, SSD는 쓰기 증폭과 마모 관리가 눈에 들어온다. 저장장치 정책은 장치의 물리적 성질에서 출발한다.

## 디스크 스케줄링

HDD에서는 헤드 이동이 크면 성능이 나빠진다. FCFS는 요청 순서대로 처리해 공정하지만 이동 거리가 길 수 있다. SSTF는 가까운 요청을 먼저 처리해 이동을 줄이지만, 멀리 있는 요청이 계속 밀릴 수 있다.

SCAN과 C-SCAN은 엘리베이터처럼 한 방향으로 이동하며 요청을 처리한다. 평균 성능과 starvation 완화 사이의 절충이다. SSD에서는 물리 헤드 이동이 없으므로 이런 알고리즘의 의미가 달라진다.

## 스왑 공간의 역할

스왑 공간은 물리 메모리가 부족할 때 페이지를 내려놓는 저장장치 영역이다. 가상 메모리와 함께 동작해 더 많은 프로세스를 실행할 수 있게 하지만, 저장장치 접근은 메모리보다 훨씬 느리다. 스왑이 과도하게 발생하면 시스템은 계산보다 I/O에 시간을 쓴다.

스왑 정책은 메모리 관리와 저장장치 관리가 만나는 지점이다. 어떤 페이지를 내보낼지, 어디에 저장할지, 다시 읽을 때 비용은 어떤지가 모두 연결된다.

## RAID는 성능과 신뢰성의 조합

RAID는 여러 디스크를 묶어 하나의 논리 장치처럼 사용하는 기술이다. striping은 데이터를 나눠 여러 디스크에 배치해 병렬성을 얻고, mirroring은 같은 데이터를 복제해 장애에 대비한다. parity는 용량 효율을 유지하면서 복구 정보를 저장한다.

RAID 레벨은 숫자만 외우기보다 목표를 봐야 한다. RAID 0은 성능을 얻지만 중복이 없고, RAID 1은 복구가 쉽지만 용량 비용이 크다. parity 기반 RAID는 용량 효율이 좋지만 쓰기 비용과 재구성 비용이 있다.

![대용량 저장장치 - RAID 기본 구조](/os-concepts/diagrams/os-11-mass-storage-structure.png)

> 그림 읽기: RAID는 striping, mirroring, parity를 조합해 성능과 복구 가능성을 조절한다. RAID 0, 1, 5의 차이는 숫자보다 중복을 어디에 두는지로 보는 편이 낫다.

## 예시: HDD와 SSD에서 같은 요청을 보낼 때

1. HDD는 헤드가 움직이고 원판이 돌아야 하므로 가까운 요청을 묶어 처리하는 효과가 크다.
2. SSD는 기계적 이동이 없어서 seek time 중심의 스케줄링 효과가 작다. 대신 병렬 채널, 쓰기 증폭, 마모 관리가 더 눈에 들어온다.
3. 저장장치 장은 알고리즘 이름보다 장치의 물리적 성질을 먼저 잡는 편이 이해가 빠르다.

## 용어 정리

### HDD

- 의미: 회전 원판과 이동 헤드로 데이터를 읽는 저장장치.
- 기능: 큰 용량을 저렴하게 제공한다.
- 주의: 탐색 시간과 회전 지연이 성능에 큰 영향을 준다.
- 키워드: `HDD`, `seek time`

### SSD/NVM

- 의미: 기계적 움직임 없이 플래시나 비휘발성 메모리로 접근하는 저장장치.
- 기능: 낮은 지연 시간과 높은 랜덤 접근 성능을 제공한다.
- 주의: 마모 관리와 쓰기 증폭을 고려해야 한다.
- 키워드: `SSD`, `NVM`

### 디스크 스케줄링

- 의미: I/O 요청 처리 순서를 조정하는 정책.
- 기능: HDD 헤드 이동과 평균 대기 시간을 줄인다.
- 주의: SSD에서는 HDD만큼 효과가 크지 않을 수 있다.
- 키워드: `SCAN`, `C-SCAN`

### 스왑 공간

- 의미: 메모리 부족 시 페이지를 저장장치에 내려놓는 공간.
- 기능: 물리 메모리 압박을 완화한다.
- 주의: 과도한 스왑은 시스템 전체를 느리게 만든다.
- 키워드: `swap`

### RAID

- 의미: 여러 디스크를 묶어 성능이나 신뢰성을 높이는 구조.
- 기능: striping, mirroring, parity로 목적을 달성한다.
- 주의: RAID는 백업이 아니다.
- 키워드: `RAID`, `parity`

## 조심할 부분

- RAID는 장애 복구를 돕지만 백업을 대체하지 않는다.
- HDD에 좋은 스케줄링 정책이 SSD에서도 같은 의미로 좋다고 볼 수 없다.
- 스왑은 메모리를 늘려 주는 마법이 아니라 느린 저장장치를 메모리 보조로 쓰는 절충이다.
