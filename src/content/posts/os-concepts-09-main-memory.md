## 먼저 볼 부분

- 스와핑과 현대 아키텍처의 주소 변환 구조

## 주소 공간을 분리하는 이유

프로그램은 자신이 0번지부터 시작하는 연속된 메모리를 쓰는 것처럼 생각한다. 이것이 논리 주소 공간이다. 실제 RAM의 위치는 물리 주소이고, 둘 사이의 변환은 MMU와 운영체제가 함께 처리한다. 이 분리 덕분에 프로세스마다 독립된 주소 공간을 제공할 수 있다.

주소 변환은 보호에도 필요하다. 어떤 프로세스가 자기 범위를 벗어난 주소에 접근하면 하드웨어와 커널이 이를 막아야 한다. 메모리 관리는 성능 기능이면서 동시에 보호 기능이다.

## 연속 할당과 단편화

초기의 단순한 방식은 프로세스 하나를 물리 메모리의 연속된 공간에 넣는 것이다. base와 limit 레지스터만 있어도 주소 변환과 보호를 구현할 수 있다. 하지만 프로세스가 들어오고 나가면 빈 공간이 여기저기 나뉘어 외부 단편화가 생긴다.

compaction으로 빈 공간을 모을 수 있지만, 실행 중인 프로세스의 메모리를 옮기려면 비용이 크고 주소 재배치가 필요하다. 그래서 현대 시스템은 고정 크기 조각으로 나누는 페이징을 기본으로 사용한다.

## 페이징을 보는 기준

페이징은 논리 메모리를 page로, 물리 메모리를 frame으로 나눈다. 논리 주소는 page number와 offset으로 나뉘고, 페이지 테이블은 page number를 frame number로 바꾼다. offset은 그대로 유지된다. 이렇게 하면 프로세스의 페이지들이 물리 메모리 곳곳에 흩어져 있어도 연속된 주소 공간처럼 보인다.

외부 단편화는 줄지만 페이지 테이블 크기와 내부 단편화가 생긴다. 페이지 크기가 커지면 테이블은 작아지지만 내부 단편화가 커지고, 페이지 크기가 작아지면 테이블과 변환 비용이 커질 수 있다.

## TLB와 다단계 페이지 테이블

메모리를 접근할 때마다 페이지 테이블을 메모리에서 읽으면 너무 느리다. TLB는 최근 주소 변환 결과를 저장하는 작은 고속 캐시다. TLB hit이면 바로 물리 주소를 만들 수 있고, miss이면 페이지 테이블을 따라가야 한다.

주소 공간이 커질수록 페이지 테이블 전체를 단순 배열로 두기 어렵다. 다단계 페이지 테이블, inverted page table 같은 구조는 메모리 사용량을 줄이기 위한 절충이다. 주소 변환은 단순 공식이 아니라 캐시와 자료구조의 조합이다.

## 예시: 가상 주소 하나가 RAM 주소가 되기까지

1. CPU가 만든 주소는 먼저 페이지 번호와 오프셋으로 나뉜다.
2. TLB에 변환 결과가 있으면 바로 프레임 번호를 얻고, 없으면 페이지 테이블을 따라간다.
3. 페이지 테이블 항목이 유효하면 프레임 번호와 오프셋을 합쳐 물리 주소가 된다. 유효하지 않다면 10장의 page fault로 이어진다.

## 용어 정리

### 논리 주소

- 의미: 프로그램이 생성하는 주소.
- 기능: 프로세스마다 독립된 주소 공간을 갖게 한다.
- 주의: 논리 주소는 실제 RAM 위치가 아니다.
- 키워드: `logical address`

### 물리 주소

- 의미: 메모리 장치가 실제로 접근하는 주소.
- 기능: MMU가 논리 주소를 변환해 만든다.
- 주의: 운영체제가 보호와 배치를 관리한다.
- 키워드: `physical address`, `MMU`

### 연속 할당

- 의미: 프로세스를 물리 메모리의 연속된 공간에 배치하는 방식.
- 기능: 단순하지만 외부 단편화가 생긴다.
- 주의: compaction은 비용이 크다.
- 키워드: `contiguous allocation`

### 페이징

- 의미: 논리 메모리와 물리 메모리를 고정 크기 조각으로 나누는 방식.
- 기능: 외부 단편화를 줄이고 비연속 배치를 가능하게 한다.
- 주의: 내부 단편화와 페이지 테이블 비용은 남는다.
- 키워드: `paging`, `page`, `frame`

### TLB

- 의미: 페이지 번호와 프레임 번호 매핑을 저장하는 빠른 캐시.
- 기능: 주소 변환 비용을 줄인다.
- 주의: 데이터 캐시가 아니라 주소 변환 캐시다.
- 키워드: `TLB`, `hit ratio`

## 조심할 부분

- 논리 주소는 프로그램이 보는 주소이고, 물리 주소는 메모리 장치가 보는 주소다.
- 페이징은 외부 단편화를 줄이지만 내부 단편화와 페이지 테이블 비용은 남긴다.
- TLB는 데이터 캐시가 아니라 주소 변환 캐시다.
