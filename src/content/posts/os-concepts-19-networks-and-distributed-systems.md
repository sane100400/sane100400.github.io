## 분산 시스템의 동기

분산 시스템은 여러 컴퓨터를 네트워크로 연결해 자원 공유, 계산 분산, 신뢰성 향상, 확장성을 얻으려는 구조다. 사용자는 하나의 서비스처럼 느끼지만, 실제로는 여러 노드가 메시지를 주고받으며 협력한다.

운영체제 관점에서는 로컬 자원만 관리하던 문제가 네트워크 너머로 확장된다. 파일, 프로세스, 이름, 인증, 장애 처리가 모두 더 복잡해진다.

![네트워크와 분산 시스템 - 분산 시스템 실패](/os-concepts/diagrams/os-19-networks-and-distributed-systems-detail.png)

> 그림 읽기: 분산 시스템의 실패는 전부 멈추는 형태만이 아니다. 요청 지연, 패킷 유실, 서버 장애가 섞이면 클라이언트는 느린 것과 실패한 것을 구분하기 어렵다.

## 통신과 프로토콜

네트워크 통신은 정해진 프로토콜에 따라 이루어진다. TCP는 연결 기반의 신뢰성 있는 바이트 스트림을 제공하고, UDP는 더 단순한 datagram 전달을 제공한다. 어떤 프로토콜을 쓸지는 신뢰성, 지연, 순서 보장, 구현 복잡도에 따라 달라진다.

운영체제는 소켓 인터페이스를 통해 네트워크 통신을 파일과 비슷한 방식으로 사용할 수 있게 한다. 하지만 네트워크는 로컬 디스크보다 실패 양상이 다양하다.

## 이름과 위치

분산 시스템에서는 사람이 기억하는 이름을 실제 위치나 주소로 바꾸는 과정이 필요하다. DNS는 도메인 이름을 IP 주소로 바꾸고, 분산 파일 시스템은 파일 이름을 서버와 객체 위치로 연결한다. 이름과 위치를 분리하면 이동, 복제, 부하 분산이 쉬워진다.

하지만 이름 서비스 자체도 장애와 일관성 문제를 가진다. 캐시는 성능을 높이지만 오래된 정보를 제공할 수 있다.

## 분산 파일 시스템

분산 파일 시스템은 원격 파일을 로컬 파일처럼 접근하게 한다. 클라이언트는 경로를 해석하고, 서버에 요청을 보내고, 결과를 캐시할 수 있다. 사용자는 편리하지만, 여러 클라이언트가 같은 파일을 수정할 때 일관성 문제가 생긴다.

분산 시스템의 가장 까다로운 점은 partial failure다. 한 노드는 살아 있고 다른 노드는 죽을 수 있으며, 네트워크만 끊길 수도 있다. 로컬 운영체제에서는 드문 종류의 불확실성이 기본 상황이 된다.

![네트워크와 분산 시스템 - 분산 파일 시스템](/os-concepts/diagrams/os-19-networks-and-distributed-systems.png)

> 그림 읽기: 분산 파일 시스템은 이름 서비스, 서버 저장소, 클라이언트 캐시가 함께 움직인다. 캐시는 빠르지만 여러 클라이언트가 같은 파일을 볼 때 일관성 문제가 생긴다.

## 예시: 분산 파일 요청이 실패했는지 느린지 모를 때

1. 클라이언트가 서버에 요청을 보냈는데 응답이 없으면 서버가 죽었는지, 네트워크가 느린지, 응답만 유실됐는지 바로 알기 어렵다.
2. 그래서 timeout, retry, idempotent operation, 캐시 무효화 같은 정책이 필요하다.
3. 분산 시스템의 어려움은 여러 노드가 있다는 사실보다 부분 실패를 다뤄야 한다는 데 있다.

## 용어 정리

### 분산 시스템

- 의미: 네트워크로 연결된 여러 컴퓨터가 협력하는 시스템.
- 기능: 자원 공유, 확장성, 신뢰성 향상을 노린다.
- 주의: 부분 실패가 가장 까다로운 점이다.
- 키워드: `distributed system`

### 이름 해석

- 의미: 사람이 쓰는 이름을 주소나 위치로 바꾸는 과정.
- 기능: 도메인 이름, 파일 경로, 서비스 이름을 실제 위치와 연결한다.
- 주의: 이름과 위치를 분리해야 이동성과 복제가 쉬워진다.
- 키워드: `naming`, `DNS`

### 프로토콜

- 의미: 통신 주체가 메시지를 주고받는 규칙.
- 기능: 오류 처리, 순서, 흐름 제어, 연결 관리를 정의한다.
- 주의: TCP와 UDP는 신뢰성 제공 방식이 다르다.
- 키워드: `protocol`, `TCP`, `UDP`

### 투명성

- 의미: 분산되어 있다는 사실을 사용자에게 덜 보이게 하는 성질.
- 기능: 위치, 복제, 장애, 이동을 숨긴다.
- 주의: 완전한 투명성은 성능과 복잡도 비용을 만든다.
- 키워드: `transparency`

### 분산 파일 시스템

- 의미: 원격 파일을 로컬 파일처럼 다루는 시스템.
- 기능: 클라이언트 캐시와 서버 저장소를 조합한다.
- 주의: 성능과 일관성 사이의 절충을 봐야 한다.
- 키워드: `DFS`, `cache`

## 조심할 부분

- 분산 시스템은 단순히 컴퓨터가 여러 대라는 뜻이 아니라, 실패와 일관성 모델이 달라지는 환경이다.
- TCP는 신뢰성 있는 스트림을 제공하지만 애플리케이션 수준 의미까지 보장하지는 않는다.
- 캐시는 분산 파일 시스템의 성능을 높이지만 일관성 비용을 만든다.
